The Problem

AI agents debugging terminal sessions need to verify each step completed correctly. Without visibility into terminal state, agents can't confirm commands executed successfully or diagnose failures.

The Solution

Frame Capture: Save terminal state as text files after every command. Each frame captures the visible terminal buffer, enabling AI agents to verify output, track progress, and debug issues at every step.

What This Demonstrates

  • ✓ Text Snapshots: Terminal state saved as .txt files for easy AI parsing
  • ✓ Timestamped: Frame naming includes elapsed time (frame_{NNNN}_{SSS.ss}.txt)
  • ✓ Sequential: Frame numbers continue across mode switches (terminal ↔ browser)
  • ✓ Automatic: Enabled by default when --output-dir is set

Usage

# Capture frames to directory (enabled automatically when -o is set)
demorec preview script.demorec --rows 30 -o ./frames

# Explicitly enable frame capture
demorec preview script.demorec --rows 30 --frames -o ./frames

# Disable frame capture even with output dir
demorec preview script.demorec --rows 30 -o ./frames --no-frames

Frame Naming Convention

ComponentFormatExample
Prefixframe_frame_
Frame Number4-digit zero-padded0001, 0002, 0015
Elapsed TimeSeconds with 2 decimals000.00, 001.25, 012.50
Extension.txt (terminal) / .png (browser).txt

Example: frame_0005_001.95.txt = Frame #5, captured at 1.95 seconds elapsed

AI Agent Use Cases

  • ✓ Step Verification: Confirm each command produced expected output
  • ✓ Error Detection: Identify when and where failures occurred
  • ✓ State Tracking: Monitor working directory, environment changes
  • ✓ Regression Testing: Compare frame sequences across runs
  • ✓ Documentation: Generate step-by-step walkthroughs from frames
